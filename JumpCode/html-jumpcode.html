🏗️ Document Structure

HTML Document Template
Every HTML document should start with this structure
<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Page Title</title> <link rel="stylesheet" href="style.css"> </head> <body> <!-- Your content here --> <script src="script.js"></script> </body> </html>
DOCTYPE: Declares the document type as HTML5
<html>: Root element of the document
<head>: Contains metadata and scripts
<body>: Contains all visible content
Meta Tags
Provide metadata about the HTML document
<meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <meta name="description" content="Page description"> <meta name="keywords" content="keywords, here"> <meta name="author" content="Author Name"> <meta name="theme-color" content="#667eea">

✨ Best Practices

Accessibility (A11y)
Make your site accessible to everyone
<!-- Always use alt text for images --> <img src="image.jpg" alt="Description of image"> <!-- Use semantic HTML --> <button>Click me</button> instead of <div onclick="..."> <!-- Use proper headings hierarchy --> <h1>Main title</h1> <h2>Subsection</h2> <!-- Label form inputs --> <label for="email">Email:</label> <input id="email" type="email"> <!-- Use ARIA attributes when needed --> <div role="alert">Important message</div>
SEO Best Practices
Optimize your site for search engines
<!-- Use descriptive titles --> <title>Specific Page Title - Site Name</title> <!-- Add meta description --> <meta name="description" content="Page description 150-160 chars"> <!-- Use semantic HTML --> <h1>Main keyword</h1> <h2>Related keywords</h2> <!-- Use descriptive alt text --> <img alt="Descriptive alt text"> <!-- Create clean URLs --> <a href="/blog/article-title">Article Title</a> <!-- Use schema.org markup --> <script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Article" } </script>
Performance Tips
Optimize page loading and performance
<!-- Optimize images --> <img src="image.jpg" loading="lazy"> <!-- Use appropriate image formats --> <picture> <source srcset="image.webp" type="image/webp"> <img src="image.jpg" alt="Fallback"> </picture> <!-- Minimize CSS --> <link rel="stylesheet" href="style.min.css"> <!-- Place scripts before closing body --> <script src="script.min.js" defer></script> <!-- Use async for non-critical scripts --> <script async src="analytics.js"></script> <!-- Preload important resources --> <link rel="preload" href="font.woff2" as="font">

💡 Pro Tips:
  • Always include DOCTYPE: <!DOCTYPE html> at the start
  • Use semantic HTML: It improves SEO, accessibility, and code readability
  • Always add alt text to images: Essential for accessibility and SEO
  • Use proper heading hierarchy: Don't skip from h1 directly to h3
  • Associate labels with inputs: Use matching id and for attributes
  • Validate your HTML: Use W3C HTML Validator
  • Use lowercase for element names: HTML is case-insensitive but lowercase is standard
  • Close all tags properly: Some are self-closing like <img>, <input>, <br>, <hr>
  • Use meaningful class and id names: Avoid names like "box1", "div2", use "hero-section", "main-nav"
  • Keep HTML semantic-focused: Use CSS for styling, not HTML attributes
